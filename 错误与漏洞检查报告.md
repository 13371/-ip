# 砚白配置IP - 错误与漏洞检查报告

## 一、已修复的问题

### 1. 配置加载健壮性（config.py）
- **问题**：若用户手动修改 `config.json` 导致 `templates` 不是数组（如 null、对象），后续遍历会报错或行为异常。
- **修复**：`load_config()` 中强制校验返回结构，保证 `templates` 始终为 list，否则置为 `[]`。

### 2. 模板缺少 id 导致崩溃（template_editor.py、main.py）
- **问题**：旧配置或手动编辑的 JSON 里模板可能没有 `"id"` 字段，访问 `t["id"]` 会 `KeyError`，列表筛选、编辑、删除时可能崩溃。
- **修复**：所有使用模板处改为 `t.get("id")` 或 `t.get("id") or ""`；列表展示时若无 id 则用 `uuid.uuid4()` 生成临时 id，避免 Treeview 报错。

### 3. 模板字段为 null 传入 netsh（main.py）
- **问题**：JSON 中若为 `"ip": null`，`template.get("ip", "")` 仍得到 `None`，拼成 `addr=None` 传给 netsh 导致异常。
- **修复**：`_apply_template()` 中对所有字段使用 `(template.get("ip") or "").strip()` 等形式，保证传入的均为字符串。

### 4. 网络参数未校验即调用 netsh（network_manager.py）
- **问题**：若通过异常配置或其它路径传入非法 IP/掩码/网关/DNS（如含特殊字符、命令片段），虽无 shell 注入风险（未用 shell=True），但可能让 netsh 报错或行为异常。
- **修复**：在 `set_static_ip()` 内对 ip、mask、gateway、dns1、dns2 做**服务端校验**：仅允许标准 IPv4 或空；任一非法则直接返回错误文案，不调用 netsh。与界面校验形成双重保障。

---

## 二、安全结论

- **命令注入**：未使用 `shell=True`，且 netsh 参数为列表形式传入，用户输入的 IP/网关等不会作为 shell 命令执行，**无命令注入漏洞**。
- **路径与配置**：配置目录基于程序/exe 所在目录，未接受用户可控路径，**无路径遍历风险**。
- **权限**：修改网络需管理员、代理仅写当前用户注册表，**权限范围合理**。

---

## 三、已知限制（非漏洞）

- **计划任务路径含双引号**：若 exe 路径本身包含 `"`，创建开机启动任务时可能异常，属极端情况，一般路径不会包含。
- **配置文件过大**：若 `config.json` 被篡改为极大文件，`json.load` 会占较多内存，建议保持配置文件仅由本程序写入，不手动放入超大内容。

---

## 四、建议

- 后续若增加「导入模板」功能，应对导入内容做与当前保存时相同的校验（如 id、必填字段、IPv4 格式等），再写入配置。
- 可定期对 `config.json` 做简单备份（如保存为 `config.json.bak`），以便损坏时恢复。

以上问题已全部在代码中修复，可直接使用当前版本。
